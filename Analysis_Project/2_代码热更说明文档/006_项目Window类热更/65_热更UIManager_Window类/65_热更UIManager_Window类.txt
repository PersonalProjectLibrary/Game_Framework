1、将Assets/RealFram/FramePlug/UIFrame，移动到Assets/Scripts里

2、删除UIFrame里的UIManager程序集（不是脚本！！！）

3、ILRuntimeManager.cs里，对m_AppDomain添加属性，方便外部脚本热更使用：
public AppDomain ILRunAppDomain {  get { return m_AppDomain; } }

4、Assets/Scripts/UGUI/Window里
HotFix.cs是热更前使用的类，这里不做处理；
另外两个Window的逻辑运行类：MenuUi.cs与LoadingUi.cs进行热更，
复制到热更工程HotFix里（注意是复制一份放过去，不是直接移过去）

注：由于创建热更工程时，选项：“将解决方案和项目放在同一个目录中”的选择不同，
导致热更脚本所在文件夹的路径可能有点不同：
不放在一个目录里对应位置：HotFixProject\HotFix\HotFix里；
放在一个目录中对应位置：或HotFixProject\HotFix里；
但可以确定复制过来的这两个脚本，是和其他热更脚本类，如TestClass.cs、TestCLRBinding.cs
是放在一起的，放在同一个文件夹里；

5、热更工程添加脚本：
打开热更工程，解决方案里，选中HotFix，右键--添加--现有项；
在打开文件夹中，选中MenuUi.cs与LoadingUi.cs进行添加；
然后给MenuUi.cs与LoadingUi.cs添加上HotFix命名空间：
namespace HotFix { public class LoadingUi : Window { ... ... } }
namespace HotFix { public class MenuUi : Window { ... ... } }

6、热更工程添加报错：
Input上下文不存在；UnityEngine.dll里无Input；

网络搜索，相关博客说明：
（1）VS工程无法识别Unity导致的；
相关博客：https://blog.csdn.net/weixin_31090403/article/details/112817035
打开Unity工程，菜单栏找到首选项：Edit--Preferences--External Tools扩展工具，
设置ExternalTools的ExternalScriptsEditor为VS；
1）Unity工程使用的脚本编译软件不是VS：
ExternalScriptsEditor没有设置为vs，直接浏览，找vs安装文件夹里的devenv.exe进行选择；
2）Unity工程使用的脚本编译软件是VS
ExternalScriptsEditor已经设置为vs了，可以先取消设置成vs，然后打开Unity里的一个cs脚本；
然后关闭cs脚本，重新打开首选项设置ExternalTools的ExtrnalScriptsEditor为VS编辑器；
再重新打开vs脚本，VS就会自动去识别脚本里的成员对象；

（2）更换dll，不同版本dll里定义的对象不同，有的里面成员多，有的少不存在Input这个命名空间；
尝试解决方法1：更换其他UnityEngine.dll
结果：试遍所有UnityEngine.dll，找到唯一一个有Input定义的一个UnityEngine.dll，
在Unity安装文件夹：Unity\Editor\2021.3.39f1c1\Editor\Data\Managed里；

替换UnityEngine.dll引用后，该方法存在问题：
与UnityEngine.CoreModule.dll冲突，多个API变量，同时两个dll里都有定义冲突；

尝试解决方法2：更换UnityEngine.CoreModule.dll，
找和UnityEngine.dll适配的UnityEngine.CoreModule.dll；
结果：试遍所有UnityEngine.CoreModule.dll，发现UnityEngine.CoreModule.dll基本都一样，
没有能和当前UnityEngine.dll适配的UnityEngine.CoreModule.dll；

尝试解决方法3_1：给报错变量，添加指定引用库；
对因UnityEngine.dll和UnityEngine.CoreModule.dll冲突的变量添加指定，
如：Debug，改为：UnityEngine.Debug；
结果：没有任何反应；方法不行；

尝试解决方法3_2：给报错变量，添加明确引用库；
因dll冲突导致很多原本正常的语句变量无法正常使用，报错；
而原本只有一个Input报错找不到，上下文中不存在；
故将UnityEngine.dll换回成其他UnityEngine.dll；
然后对Input进行处理；
1）根据提示，引用UnityEngine.Window.dll，
添加：using UnityEngine.Window;
或写成UnityEngine.Window.Input；
都不能解决报错；
2）改成UnityEngine.Input，提示UnityEngine.dll里不存在，写法错误；

（3）新版本Unity，将UnityEngine.dll里内容进行拆分，当前UnityEngine.dll是空的；
相关博客：https://blog.51cto.com/u_15077561/3805429
需要找到拆分出来的相关包含Input对象的UnityEngine.dll的子dll进行引用；
在前面Input改为UnityEngine.Input后，提示：
未能在UnityEngine中找到类型名"Input"。
此类型已转发到UnityEngine.InputLegacyModule；
请考虑添加对该程序集的引用；

故这里添加UnityEngine.InputLegacyModule.dll的引用；成功解决该问题；

此问题的解决过程，可查看该文本同文件夹里的“热更工程dll使用报错和解决”文件夹；
